# 클래스란?
* 클래스란 설계도임 가장 큰틀 안에 메서드,변수등을 넣음
  
# string class란
* 자바는 기본적으로 변수를 상자에 넣어서 저장하는 구조임 하지만 String 문자는 불변함 
* ` s = s+"Hello"` -> 새로운 문자의 값을 넣는것임
    * 새로운 객체를 만들어서 화살표를 표시하는 개념
# 필드선언
* 객체마다 다른값들을 저장할수있음 객체들의 데이터(속성)들을 정의함
# 메서드란
* 메서드는 클래스는 틀이라명 그 안에 있는 행동,연산등을 집합해서 만든것임
* 메서드 안에서 선언한 변수들은 메서드가 끝나면 사라짐
# 스코프와static
* 스코프는 지역변수들을 의미함 그 메서드안에서 선언하여 발생한 지역을 말함
* 하지만 메서드가 끝나도 사라지지않게 정의한게 static임 static으로 선언한 변수는 그 메서드가 끝나도 사라지지 않음

# 생성자

  * 생성자에 아무 값도 넣지않는 다면 기본 생성자가 만들어 지면서 초기의 값에 아무것도 안넣을수 있음 
클래스에 이미 생성자를 명시해 놨다면 기본 생성자는 만들어질수 없음 아무것도 생성자에 안넣으면 오류가 남

**내부 클래스에는 Static 변수를 써봤자 static의 역할을 하지않음**
-해결책-
   내부 클래스를 static으로 만듦
- 삭제
```java
package approach;

public class PublicExam {
    public static class IN_PUBLICEXAM{  // 내부 클래스에 static 변수를 쓰기 위해 static 클래스 생성
        public static int public_value;  // static 변수 선언  

    }

}
```

```java
package approach;

public class approach {
    PublicExam publicExam = new PublicExam();
     int a = PublicExam.IN_PUBLICEXAM.public_value;  // static은 무조건 클래스명으로 접근해야함(레퍼런스명 X)
}
```
- +A static은 객체를 생성하지않고 쓸수있게 하는것임,Public은 누구만 접근할거이냐

# 오류발생
```java
try{
                int k = i / j;
                System.out.println(k);
            }catch(ArithmeticException e){
                System.out.println("0으로 나눌 수 없습니다. : " + e.toString());
            }finally {
                System.out.println("오류가 발생하든 안하든 무조건 실행되는 블록입니다.");
            }
```
* `try`-> 실행 코드
* `IF Try`에 오류가 발생 -> `catch`문 실행
* `finallyd은 무조건 실행`

# 사용자 정의 Exception
```java
class MyException extends Exception {
    public MyException(String message) {
        super(message);
    }
}

void check(int age) throws MyException {
    if (age < 0) {
        throw new MyException("나이는 음수 안됨");
    }
}
```
* Throws로 오류를 발생시킨뒤 클래스로 오류를 처리하는 개념임
